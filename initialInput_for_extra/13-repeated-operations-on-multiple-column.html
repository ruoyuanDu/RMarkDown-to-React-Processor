Repeat Operations Across Multiple Columns

In the earlier section, you have learned different selection helper functions that allow efficient column selection. In this tutorial, you’ll learn another powerful and advanced selection helper function across() that enables you to repeat the same operations across multiple columns at the same time.

Basics of repeated operations across columns

Often you want to apply a processing repeatedly over multiple columns. In the following example, n_distinct() is verbosely typed 3 times to count unique entries in each of the three columns, “name”, “hair_color, and”skin_color”.

library(dplyr)
# a subset of the first 5 columns
starwars2 <- starwars[, 1:5] 

starwars2 %>% 
  summarise(name       = n_distinct( name       ),
            hair_color = n_distinct( hair_color ),
            skin_color = n_distinct( skin_color ))
  # A tibble: 1 × 3
     name hair_color skin_color
    <int>      <int>      <int>
  1    87         12         31

The code can be much simplified using the across() function. Inside this function, the names of columns to be processed are fed to the first argument .col, and the function name (without carrying its parentheses) is fed to the second argument .fns. Note that both arguments are inside the across() function.

starwars2 %>% 
  summarise(across(.col = c(name, hair_color, skin_color), 
                   .fns = n_distinct))
  # A tibble: 1 × 3
     name hair_color skin_color
    <int>      <int>      <int>
  1    87         12         31

You can use other helper functions inside across() to select columns. (Below the argument names of across() are omitted.)

# count unique values separately for each character variable
starwars2 %>% 
  summarise(across(where(is.character), n_distinct))
  # A tibble: 1 × 3
     name hair_color skin_color
    <int>      <int>      <int>
  1    87         12         31

The following example attempts to calculate the mean of all numeric columns. Due to the presence of NA values, the mean is output as NA.

starwars2 %>% 
  summarise(across(where(is.numeric), mean))
  # A tibble: 1 × 2
    height  mass
     <dbl> <dbl>
  1     NA    NA

To include additional arguments of the mean() function (e.g., na.rm = T to remove the NA values), use the purrr style with the tilde operator ~ to create an anonymous function in the form of ~function(.x, ...).

# use the purrr style to add more arguments of the mean function
starwars2 %>% 
  summarise(across(where(is.numeric), ~mean(.x, na.rm = T)))
  # A tibble: 1 × 2
    height  mass
     <dbl> <dbl>
  1   175.  97.3

The argument .names specifies the names of the output columns using the glue syntax. The default value is {.col}; that is, it keeps the original column names. The following example adds “_mean” suffix to the output column names.

starwars2 %>% summarise(across(
  where(is.numeric), ~mean(.x, na.rm = T), 
  .names = "{.col}_mean"))
  # A tibble: 1 × 2
    height_mean mass_mean
          <dbl>     <dbl>
  1        175.      97.3

The grouping variables (specified inside group_by) are not selected by across(). In the example below, unique values are counted in all character columns separately for each kind of hair color (the grouping variable); although “hair_color” is also a character variable, it is not included in across().

starwars2 %>% 
  group_by(hair_color) %>% 
  summarise(across(where(is.character), n_distinct))
  # A tibble: 12 × 3
    hair_color     name skin_color
    <chr>         <int>      <int>
  1 auburn            1          1
  2 auburn, grey      1          1
  3 auburn, white     1          1
  4 black            13          7
  # ℹ 8 more rows

Operate multiple functions across columns

To perform multiple functions repeatedly over a same set of columns, a list of functions can be fed into the function argument .fns. The example below calculates the mean, standard deviation, and number of missing values NA in each of the numeric columns.

f <- list(
  m = ~mean(.x, na.rm = T), # calculate mean
  s = ~sd(.x, na.rm = T),   # calculate standard deviation
  n = ~sum(is.na(.x))       # count number of missing values
)

starwars2 %>% summarise(across(where(is.numeric), f))
  # A tibble: 1 × 6
    height_m height_s height_n mass_m mass_s mass_n
       <dbl>    <dbl>    <int>  <dbl>  <dbl>  <int>
  1     175.     34.8        6   97.3   169.     28

In case of the listed function operation, .names defaults to {.col}_{.fn}, i.e., it names new columns with a combination of the original column names and the function names. The following script swaps the naming order (function name first, column name second), and connect them with a hyphen.

starwars2 %>% summarise(
  across(where(is.numeric), f, .names = "{.fn}-{.col}"))
  # A tibble: 1 × 6
    `m-height` `s-height` `n-height` `m-mass` `s-mass` `n-mass`
         <dbl>      <dbl>      <int>    <dbl>    <dbl>    <int>
  1       175.       34.8          6     97.3     169.       28

By default, columns produced from the same variables are clustered together. Alternatively, you can use relocate() to cluster columns produced from the same function, and use selection helpers to select blocks of columns.

starwars2 %>% summarise(
  across(where(is.numeric), f, .names = "{.fn}-{.col}")) %>% 
  # start with mean columns, then standard deviation columns
  relocate(contains("m-"), contains("s-"))
  # A tibble: 1 × 6
    `m-height` `m-mass` `s-height` `s-mass` `n-height` `n-mass`
         <dbl>    <dbl>      <dbl>    <dbl>      <int>    <int>
  1       175.     97.3       34.8     169.          6       28

Use with other dplyr functions

🎁 Bonus skill

The use of across() with mutate() and other dplyr verbs works the same way. The following example scales all numeric variables to a range of 0 to 1.

# define function scaling a numeric vector to range of [0, 1]
rescale_0_1 <- function(x) {
  minim <- min(x, na.rm = T)
  maxim <- max(x, na.rm = T)
  (x - minim) / (maxim - minim)
}

# overwrite the original columns with scaled values
starwars2 %>% mutate(across(where(is.numeric), rescale_0_1))
  # A tibble: 87 × 5
    name           height   mass hair_color skin_color 
    <chr>           <dbl>  <dbl> <chr>      <chr>      
  1 Luke Skywalker  0.535 0.0462 blond      fair       
  2 C-3PO           0.510 0.0447 <NA>       gold       
  3 R2-D2           0.152 0.0127 <NA>       white, blue
  4 Darth Vader     0.687 0.0901 none       white      
  # ℹ 83 more rows
# keep original columns,
# include scaled values as new columns with suffix "_scaled"
starwars2 %>% mutate(across(
  where(is.numeric), rescale_0_1, .names = "{.col}_scaled"),
  # put new columns after the "mass" column
  .after = mass)
  # A tibble: 87 × 7
    name           height  mass height_scaled mass_scaled hair_color skin_color 
    <chr>           <int> <dbl>         <dbl>       <dbl> <chr>      <chr>      
  1 Luke Skywalker    172    77         0.535      0.0462 blond      fair       
  2 C-3PO             167    75         0.510      0.0447 <NA>       gold       
  3 R2-D2              96    32         0.152      0.0127 <NA>       white, blue
  4 Darth Vader       202   136         0.687      0.0901 none       white      
  # ℹ 83 more rows
