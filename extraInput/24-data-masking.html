Data Masking

What is data masking?

Data masking is a special feature used in many dplyr functions, such as arrange(), count(), filter(), group_by(), mutate(), and summarise() (these functions are marked with <data-masking> in their R documents). Data masking allows you to refer to variables (columns) in a data frame as if they were standalone variables (objects) in an environment, without explicitly using the data$var syntax, thus making data wrangling easier with less typing.

Consider the following script. It involves multiple typing of starwars2$ in order to access the column values.

library(dplyr)
starwars2 <- starwars[, 1:5]
starwars2[starwars2$height > 100 & starwars$skin_color == "gold", ]
  # A tibble: 1 × 5
    name  height  mass hair_color skin_color
    <chr>  <int> <dbl> <chr>      <chr>     
  1 C-3PO    167    75 <NA>       gold

Data masking makes the script more succinct and readable.

starwars2 %>% filter(height > 100 & skin_color == "gold")

The above example presents two types of “variables”:

  • Environment-variable (e.g., starwars and starwars2) are “programming” variables that are stored in an (e.g., global) environment. They are created by importing an external file, by assignment via <-, or exist as R packages’ built-in dataset (as the case here).

  • data-variables (e.g., height and skin_color) are “statistical” variables that are stored in a dataset. In the above example, using data masking, you can access these variables by directly calling the variable (column) names, the same way you call a variable (object) from an environment.

Data masking in programming

While data masking makes it easier to access data variables in routine data wrangling, it requires a bit extra syntax notation when used in programming. There requires two separate syntax for accessing data variables and environment variables.

Access data-variable via { }

The following script creates a function to group a dataset based on one variable, and then calculate the mean of a second variable for each group. Here var1 and var2 are treated as data-variables. While defining the function, they are accessed in group_by() and summarise() with doubled curly braces { var }.

func_uniqueLevels <- function(mydata, var1, var2){
  mydata %>% 
    # wrap var1 and var2 in '{{ }}' 
    group_by({{ var1 }}) %>% 
    summarise(mean = mean({{ var2 }}, na.rm = T ))
}

starwars2 %>% 
  # calculate height for each type of hair color
  func_uniqueLevels(var1 = hair_color, var2 = height)
  # A tibble: 12 × 2
    hair_color     mean
    <chr>         <dbl>
  1 auburn         150 
  2 auburn, grey   180 
  3 auburn, white  182 
  4 black          174.
  # ℹ 8 more rows

Access environment-variable via .data[[ ]]

Data masking can be equivalently written with the .data[[ "var" ]] syntax: it includes a fixed .data pronoun (it’s not a dataset), the doubled square brackets, and quoted data-variables. For instance, the following two lines are equivalent:

starwars2 %>% filter(height > 10)
starwars2 %>% filter(.data[["height"]] > 10)

The same syntax can be used when defining a function with access to a environment-variable. For instance, the earlier created function func_uniqueLevels is equivalent to the code below. Here var1 and var2 are defined as environment-variables (e.g., var1 = "hair_color"), i.e., objects available in the local environment of the function func_uniqueLevels . Therefore, they are accessed in group_by() and summarise() with the .data[[var]] syntax.

func_uniqueLevels <- function(mydata, var1, var2){
  mydata %>% 
    # wrap var1 and var2 in .data[[...]]
    group_by(.data[[var1]] ) %>% 
    summarise(mean = mean(.data[[var2]], na.rm = T ))
}

starwars2 %>% 
  # Note! the data-variables are specified as quoted strings
  func_uniqueLevels(var1 = "hair_color", var2 = "height")

Naming of columns

A newly generated column can be named programmatically by using := instead of =. Similar to discussion above, the naming syntax depends on either the data-variable or environment-variable is being accessed.

Name after data-variables with { }

If the name of a newly created column is derived from data-variables, wrap the column name in double curly brackets { } and in quote. The function below names the new column as the original column names connected with a hyphen.

func.paste <- function(mydata, var1, var2){
  mydata %>% 
    # new column name in double curly brackets
    mutate("{{var1}}-{{var2}}" := paste({{var1}}, "-", {{var2}}),
           .before = 2) # new column as the 2nd column
}

# Note! The new column is output as the 2nd column
starwars2 %>% func.paste(var1 = height, var2 = mass)
  # A tibble: 87 × 6
    name           `height-mass` height  mass hair_color skin_color 
    <chr>          <chr>          <int> <dbl> <chr>      <chr>      
  1 Luke Skywalker 172 - 77         172    77 blond      fair       
  2 C-3PO          167 - 75         167    75 <NA>       gold       
  3 R2-D2          96 - 32           96    32 <NA>       white, blue
  4 Darth Vader    202 - 136        202   136 none       white      
  # ℹ 83 more rows

Name after environemnt-variables using the glue syntax { }

If the new column’s name is derived from an environment variable, you can use the glue syntax (with a single pair of curly brackets { }). The immediately above example is equivalent to the code below. The new column name is based on var1 and var2 that are defined variables / objects available in the local function environment.

func.paste <- function(mydata, var1, var2){
  mydata %>% 
    # new column name in SINGLE pair of curly brackets
    mutate("{var1}-{var2}" := paste(.data[[var1]], "-", .data[[var2]])) 
}

# column names supplied as quoted strings
starwars2 %>% func.paste(var1 = "height", var2 = "mass")
