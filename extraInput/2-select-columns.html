Select Columns from a Dataset

This tutorial explains how to select columns using the select() function. In addition, a range of related techniques will be demonstrated, which are helpful not only for select(), but also for many other functions of dplyr.

This tutorial covers the following four topics:

  • Basics of column selection
  • Selection helper functions
  • Select with operators
  • Select in purrr style

Dataset

We’ll use a subset (first five columns) of the starwars dataset for demonstration with ease of learning. The starwars dataset is built in the dplyr package.

library(dplyr)
starwars2 <- starwars[, 1:5] # select first five columns

Basics of column selection

Select columns with select(). Note that the column names are not quoted; you access the column name as if you are calling the name of an object or variable (a technique known as data masking that makes typing easier for routine tasks).

starwars2 %>% select(name, height, skin_color)
  # A tibble: 87 × 3
    name           height skin_color 
    <chr>           <int> <chr>      
  1 Luke Skywalker    172 fair       
  2 C-3PO             167 gold       
  3 R2-D2              96 white, blue
  4 Darth Vader       202 white      
  # ℹ 83 more rows

Equivalently, you can also select column names inside the concatenation function c(), or use the corresponding column index number.

# put column names inside c() 
starwars2 %>% select(c(name, height, skin_color))

# select 1st, 2nd and 5th column
starwars2 %>% select(c(1, 2, 5))

Use the colon operator : to select consecutive columns.

# select consecutive columns from "name" to "mass"
starwars2 %>% select(name:mass)
  # A tibble: 87 × 3
    name           height  mass
    <chr>           <int> <dbl>
  1 Luke Skywalker    172    77
  2 C-3PO             167    75
  3 R2-D2              96    32
  4 Darth Vader       202   136
  # ℹ 83 more rows

Use the negative operator - to remove specified columns from the dataset.

# Remove "name" and "mass" columns. 
starwars2 %>% select(-c(name, mass))
  # A tibble: 87 × 3
    height hair_color skin_color 
     <int> <chr>      <chr>      
  1    172 blond      fair       
  2    167 <NA>       gold       
  3     96 <NA>       white, blue
  4    202 none       white      
  # ℹ 83 more rows

In the above example, you can also use the logic negation operator ! to exclude columns. However, the minus - and exclamation ! are not always exchangeable (as in this case).

Improve selection efficiency with helpers functions

Use starts_with() to select columns whose header names start with the specified prefix, and ends_with() to select columns ended with the specified suffix.

# select column "name" and columns with names ended with "color“
starwars2 %>% select(name, ends_with("color"))
  # A tibble: 87 × 3
    name           hair_color skin_color 
    <chr>          <chr>      <chr>      
  1 Luke Skywalker blond      fair       
  2 C-3PO          <NA>       gold       
  3 R2-D2          <NA>       white, blue
  4 Darth Vader    none       white      
  # ℹ 83 more rows

Use contains() to select columns whose names contain the specified strings.

# select all columns whose name contain "eight"
starwars2 %>% select(contains("eight"))
  # A tibble: 87 × 1
    height
     <int>
  1    172
  2    167
  3     96
  4    202
  # ℹ 83 more rows

Use matches() to match column names using regular expression. In the following example, we use character class to specify the search pattern.

# select all columns with names containing the letter "n" or "m". 
starwars2 %>% select(matches("[nm]"))
  # A tibble: 87 × 3
    name            mass skin_color 
    <chr>          <dbl> <chr>      
  1 Luke Skywalker    77 fair       
  2 C-3PO             75 gold       
  3 R2-D2             32 white, blue
  4 Darth Vader      136 white      
  # ℹ 83 more rows

where() takes a function as its input. This input function returns a TRUE or FALSE for each column, and columns judged as TRUE will be selected. Note that inside where(), the function does not carry any parenthesis. In the following example, we select columns that are a specific type; similar functions include is.factor(), is.numeric(), and is.integer(), etc.

# test if a column is a "character" variable
# the output of the function is a "logic" vector 
is.character(starwars2$name)
  [1] TRUE
# select only "character" columns 
# note that there are no parentheses in the input function
starwars2 %>% select(where(is.character))
  # A tibble: 87 × 3
    name           hair_color skin_color 
    <chr>          <chr>      <chr>      
  1 Luke Skywalker blond      fair       
  2 C-3PO          <NA>       gold       
  3 R2-D2          <NA>       white, blue
  4 Darth Vader    none       white      
  # ℹ 83 more rows

Select with operators

Logical operators such as AND &, OR | , and negation !, are frequently used in select() and filter(). Below are several examples demonstrating their use.

  • Select columns that are simultaneously (1) “character” variables, and (2) ended with “color” in the column name. Both criteria must be met, as noted by the intersection operator &.
starwars2 %>% 
  select(where(is.character) & ends_with("color"))
  # A tibble: 87 × 2
    hair_color skin_color 
    <chr>      <chr>      
  1 blond      fair       
  2 <NA>       gold       
  3 <NA>       white, blue
  4 none       white      
  # ℹ 83 more rows
  • Select columns that are either a “numeric” variable(including both the “integer” and “double” type), or contains “skin” in the column name, noted by the union (“or”) operator |.
starwars2 %>% 
  select(where(is.numeric) | contains("skin"))
  # A tibble: 87 × 3
    height  mass skin_color 
     <int> <dbl> <chr>      
  1    172    77 fair       
  2    167    75 gold       
  3     96    32 white, blue
  4    202   136 white      
  # ℹ 83 more rows
  • Rearrange the order of columns: first select column “name”, and then columns ended with “color”, and finally those not ended with “color”. Note that the NOT is expressed via the logic negation operator !, instead of the column removal operator -. In some cases though these two operators are exchangeable, as in this example.
starwars2 %>% 
  select(name, ends_with("color"), !ends_with("color"))
  # A tibble: 87 × 5
    name           hair_color skin_color  height  mass
    <chr>          <chr>      <chr>        <int> <dbl>
  1 Luke Skywalker blond      fair           172    77
  2 C-3PO          <NA>       gold           167    75
  3 R2-D2          <NA>       white, blue     96    32
  4 Darth Vader    none       white          202   136
  # ℹ 83 more rows

Select in purrr style with extra flexibility

🏆 expert level skill

You can write where() in a purrr style for more efficient feature selection. Consider the following script.

starwars2 %>% select(where(is.numeric))
  # A tibble: 87 × 2
    height  mass
     <int> <dbl>
  1    172    77
  2    167    75
  3     96    32
  4    202   136
  # ℹ 83 more rows

The above line can be equivalently written with an anonymous function (lambda function) defined on the fly, that checks whether each column (noted as x in the anonymous function) is numeric.

starwars2 %>% 
  select(where( function(x) is.numeric(x) ))

Alternatively, you can use tilde ~ to create the anonymous function, where each column is represented by .x.

starwars2 %>% 
  select(where(~ is.numeric(.x) ))
  # A tibble: 87 × 2
    height  mass
     <int> <dbl>
  1    172    77
  2    167    75
  3     96    32
  4    202   136
  # ℹ 83 more rows

This format allows for flexible addition of more selection criteria. In the script below, numeric columns are selected only if the selected variable has a mean value above 100. && is used to connect the two criteria. (use na.rm = T to remove any NA values when calculating the mean)

starwars2 %>% select(
  name,
  where(~ is.numeric(.x) && mean(.x, na.rm = T) > 100))
  # A tibble: 87 × 2
    name           height
    <chr>           <int>
  1 Luke Skywalker    172
  2 C-3PO             167
  3 R2-D2              96
  4 Darth Vader       202
  # ℹ 83 more rows

In addition to the selection helpers mentioned above, across() is another powerful helper function that enables repeated operations across multiple columns, and is frequently used together with mutate() and summarize(). We’ll introduce this more advanced technique in a later section.
